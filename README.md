# FSD Next.js Tailwind Template

Стартовый шаблон для Next.js приложения с архитектурой Feature Sliced Design и Tailwind CSS.

## Технологии

- **Next.js 16** — React-фреймворк с серверным рендерингом
- **React 19** — библиотека для создания пользовательских интерфейсов
- **TypeScript** — типизированный JavaScript
- **Tailwind CSS 4** — утилитарный CSS-фреймворк
- **ESLint + Prettier** — линтинг и форматирование кода

## Архитектура: Feature Sliced Design

Проект организован по методологии FSD — это масштабируемая и модульная архитектура для фронтенд-приложений.

```
src/
├── 1_app/           # Точка входа, конфигурация, стили
│   ├── layouts/     # Layout-компоненты
│   └── styles/      # Глобальные стили, шрифты
├── 2_pages/         # Страницы (маршруты приложения)
│   └── home/        # Пример страницы Home
├── 3_widgets/       # Комплексные блоки UI
├── 4_features/       # Бизнес-логика фич
├── 5_entities/      # Сущности предметной области
└── 6_shared/        # Переиспользуемые компоненты, утилиты
```

### Уровни FSD

| Уровень      | Назначение                                      |
| ------------ | ----------------------------------------------- |
| `1_app`      | Точка входа, конфигурация приложения            |
| `2_pages`    | Страницы — маршруты приложения                  |
| `3_widgets`  | Widgets — композиция фич в блоки                |
| `4_features` | Features — законченные бизнес-функции           |
| `5_entities` | Entities — сущности домена (User, Product)      |
| `6_shared`   | Shared — переиспользуемый код (UI-kit, утилиты) |

### Правила импорта

- Импорты идут только от нижних уровней к верхним
- Нельзя импортировать из `3_widgets` в `4_features`
- Shared-компоненты доступны везде

## Быстрый старт

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Сборка проекта
npm run build

# Запуск линтера
npm run lint
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## Структура страницы

Страницы Next.js находятся в `app/`, а FSD-компоненты — в `src/`. Пример подключения:

```tsx
// app/page.tsx
import { Page as default, metadata } from "@/2_pages/home"
```
